## はじめに

俺はプログラマーじゃない。コーディングは趣味で、設計して、AIに実装を任せて、レビューするスタイルだ。言語は気にしていない。AIが書いてくれるので。

そんな人間が、AIコーディングツールを使って**画面翻訳アプリを作り、BOOTHで販売するところまで持っていった**話を書く。

---

## 何を作ったか

**OLTranslator**（オーエル・トランスレーター）。Windows用のリアルタイム画面翻訳アプリだ。

やることはシンプル。画面上の好きな範囲を選択すると、そこに映っている外国語テキストをOCRで読み取り、日本語訳をその場にオーバーレイ表示する。

### 主な特徴

- **ローカルOCR** — RapidOCR + ONNX Runtimeで画面上のテキストを認識。ネット不要
- **翻訳はローカルでもオンラインでも** — 同梱のローカル翻訳（Argos）がデフォルト。Google Cloud Translation、DeepL、Azure、Amazonにも対応
- **オーバーレイ表示** — 翻訳結果が元の文字位置に重なって表示される。半透明背景で、クリックは透過するので操作の邪魔にならない
- **英語・韓国語対応** — ソース言語を切り替えられる
- **常駐型** — システムトレイに入って、選択範囲を記憶。次回起動時に自動再開

### 使い方

1. アプリを起動
2. 翻訳したい画面範囲をドラッグで選択
3. あとは自動。OCRと翻訳が繰り返されて、結果がオーバーレイ表示される

ゲーム、配信、ドキュメント、何でもいい。画面に外国語が映っていれば、そこを日本語にしてくれる。

## どうやって作ったか

VS Code + GitHub Copilotで開発して、完成まで約2週間。AIに投げまくっただけだ。

OCR部分、翻訳API連携、オーバーレイ表示、設定画面、システムトレイ、インストーラー。全部AIに実装を任せて、自分は設計と方針決めとレビューに集中した。

もちろん、すべてがスムーズだったわけじゃない。オーバーレイのちらつき、OCRのノイズ除去、Program Files配下での権限問題。一つずつAIと潰していった。複雑なバグの追跡にはClaude Codeも使った。文脈を粘り強く追いかけてくれるので、原因の切り分けが速い。

## AIコーディングで「売れるもの」は作れるか

結論から言えば、**作れる**。

俺はプログラマーじゃない。C++を手で書いたわけでもない。でもOCR、翻訳API、オーバーレイ表示、インストーラー、ライセンス認証まで含めた製品レベルのアプリが出来た。

大事だったのは：

- **何を作るか明確にすること** — AIは「いい感じに作って」では動かない。仕様を自分で決める
- **設計を自分でやること** — AIにアーキテクチャ任せると破綻する。構成は自分で決めて、実装を任せる
- **レビューを怠らないこと** — AIが書いたコードをそのまま信用しない。動かして、確認して、おかしければ戻す

AIコーディングは「コードを書けない人がコードを書ける」ようになる魔法じゃない。**「設計できる人が、実装のスピードを10倍にできる」ツール**だ。
